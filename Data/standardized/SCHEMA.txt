================================================================================
STANDARDIZED (TIDY) DATA — SCHEMA
Tumbly paper : Lab A + Lab B datasets combined into one long-format schema
================================================================================
Every table carries a `dataset` column ("Lab A" or "Lab B") so the two
studies stack into a single frame for cross-dataset figures. All tables are
tidy/long (one observation per row). Built by scratchpad/standardize.py from the
header-cleaned CSVs in ../Lab A and ../Lab B. Regenerating overwrites these.

--------------------------------------------------------------------------------
SHARED CONVENTIONS
--------------------------------------------------------------------------------
- date        : ISO YYYY-MM-DD.
- day_index   : integer days relative to each study's TRF start (= day 0):
                  Lab A TRF start = 2026-06-25
                  Lab B TRF start = 2026-06-20
                Negative = before TRF (ADL/AL or acclimation). This is the
                natural shared x-axis for aligning the two studies.
- diet        : unified 2-level factor for TRF-vs-control plots:
                  "TRF"     = time-restricted feeding
                  "control" = ad libitum (Lab A "ADL" / Lab B "AL")
                (In subjects.csv, Lab A mice are "within (ADL+TRF)" because the
                 same mouse is measured under BOTH conditions.)
- phase       : original, more detailed label (ADL / AL / TRF / baseline /
                transition). Use this when you need the raw study labels.
- design      : "within"  = Lab A (paired; each mouse is its own control)
                "between" = Lab B (independent TRF vs AL groups)
- subject_type: "mouse" or "cage" (Lab B food is measured per cage, not per
                mouse — see caveats).

--------------------------------------------------------------------------------
subjects.csv    (28 rows: 6 Lab A mice + 22 Lab B mice)
--------------------------------------------------------------------------------
One row per animal. Join key = (dataset, subject_id).
  dataset, subject_id, subject_type, diet, design, sex, strain_background,
  age, cage, tumbly_unit, future_condition, notes
- Lab A: subject_id = numeric ear tag (1316,1338-1342); cage/unit = NA.
- Lab B: subject_id = E1..K4; cage = DLAR cage # (4743-4777);
           future_condition = the eventual injury/sham+diet label (NOT applied
           in this dataset); tumbly_unit = NA (see caveats).

--------------------------------------------------------------------------------
body_weight_long.csv    (162 rows)
--------------------------------------------------------------------------------
  dataset, subject_id, subject_type, diet, design, phase, date, day_index,
  body_weight_g
- Lab A: 6 mice x 16 days (baseline, 7 ADL, transition, 7 TRF).
- Lab B: 22 mice x 3 pre-injury weigh-ins (2026-06-20, -23, -26).

--------------------------------------------------------------------------------
food_intake_long.csv    (153 rows)   value = grams / mouse / day
--------------------------------------------------------------------------------
  dataset, subject_id, subject_type, diet, design, phase, date, day_index,
  food_g_per_mouse_per_day
- Lab A: per-MOUSE (subject_type=mouse), 6 mice x 15 days.
- Lab B: per-CAGE  (subject_type=cage), 7 cages x 9 days (Day -1..7).
           subject_id = cage letter E..K.

--------------------------------------------------------------------------------
device_summary_long.csv    (222 rows)   one row per unit per day
--------------------------------------------------------------------------------
  dataset, unit, date, n_samples, mean_battery_v, min_battery_v, pct_error,
  pct_door_open, n_timeddoor, n_freefeeding, day_index
- Aggregated from the raw ~30 s Tumbly logs.
- pct_error     : % of samples whose Error != "OK".
- pct_door_open : % of samples with DoorOpen == 1 (≈33% on TRF days = 8 h window;
                  ≈100% on free-feeding days).
- unit          : Lab A = TUMBLYnnn (device number); Lab B = Unit5/6/9/10.
- Lab B has TRF units only (AL cages are untracked — door left open).

--------------------------------------------------------------------------------
device_events_long.csv.gz    (566,994 rows)   the RAW per-sample event stream
--------------------------------------------------------------------------------
  dataset, unit, datetime, date, day_index, time_of_day_h, task, battery_v,
  light_sensor, door_open, servo_feedback, error, is_error
- Every ~30 s Tumbly log line from all units, concatenated and normalized.
  Full resolution — use this to plot door state, errors, battery, servo over time.
- gzip-compressed (4.4 MB); pandas & raw-GitHub read it directly:
      pd.read_csv(".../device_events_long.csv.gz")
- time_of_day_h : hours since midnight (float) — handy for feeding-window plots.
- is_error      : True when error != "OK". Built by build_device_events.py.
- Row count == sum(n_samples) in device_summary_long.csv (completeness verified).

--------------------------------------------------------------------------------
door_transitions_long.csv    (314 rows)   door open/close events only
--------------------------------------------------------------------------------
  dataset, unit, datetime, date, day_index, time_of_day_h, transition, task
- One row per door state change (transition = "open" 0->1 or "close" 1->0),
  derived from device_events_long. Compact table for open/close-time figures.
- Observed: Lab B opens ~17:00 / closes ~01:00; Lab A opens ~20:00 /
  closes ~04:00 (matches the protocols in each METHODS.txt).

--------------------------------------------------------------------------------
CAVEATS / ASSUMPTIONS (important for interpretation)
--------------------------------------------------------------------------------
1. Designs are NOT interchangeable: Lab A is within-subject/paired (n=6,
   ~8-mo APOE4); Lab B is between-group (TRF n=12 / AL n=10, ~7-8-wk C57BL/6J).
   Keep `design` in mind when combining or running statistics.
2. Lab B FOOD is per-cage (hopper weight / # mice), not per individual mouse;
   Lab A food is per individual mouse. Do not treat Lab B food subjects as
   independent mice.
3. Lab B food cage labels were recovered by mapping the Prism group-replicate
   columns to cages (TRF_1..4 -> E,F,I,J ; AL_1..3 -> G,H,K) and were verified
   against the food-hopper spreadsheet. High confidence, but an inferred mapping.
4. tumbly_unit per mouse/cage = NA: the 4 Lab B TRF units (5,6,9,10) correspond
   to the 4 TRF cages (E,F,I,J) but the exact unit->cage pairing is not in the
   source data.
5. Circadian/actogram data exists for Lab A ONLY; it is not part of this tidy
   set (see ../Lab A ClockLab CSVs). Prism-computed statistics tables are also
   excluded here by design — recompute stats from these tidy tables for
   reproducibility.
6. Feeding windows differ (Lab A 8PM-4AM dark-phase; Lab B 5PM-1AM EST) and
   are recorded in each folder's METHODS.txt, not forced to match.
